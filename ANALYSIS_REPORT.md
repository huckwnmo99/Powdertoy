## 3. 증류탑 서바이벌 게임 - 상세 분석

### 3.1 게임 컨셉

화학 플랜트의 **증류탑(Distillation Column)** 운전을 시뮬레이션한 서바이벌 게임이다. 플레이어는 **HMI(Human-Machine Interface)** 화면을 통해 증류탑의 온도와 압력을 안전 범위 내로 유지해야 한다. 온도 또는 압력이 한계치(0 또는 100)에 도달하면 게임 오버된다.

### 3.2 UI 구조 (index.html)

```
┌─────────────────────────────────────────────┐
│  DISTILLATION COLUMN HMI v2.0   [LED] 00:00 │  ← 헤더 (로고, 알람 LED, 점수, 일시정지)
├────────┬──────────────┬─────────────────────┤
│ Tower  │   Column     │  Tower              │
│ Temp   │  (증류탑     │  Press              │  ← 메인 작업 영역
│ TR-101 │   그래픽)    │  PI-102             │     (좌: 온도게이지, 중: 증류탑, 우: 압력게이지)
│ [게이지]│             │  [게이지]            │
├────────┴──────────────┴─────────────────────┤
│  ♨️ Reboiler Steam    │   ❄️ Reflux Flow     │  ← 제어 패널
│  [======슬라이더======]│  [======슬라이더====]│     (스팀 밸브, 리플럭스 밸브)
├─────────────────────────────────────────────┤
│       🧪 The Powder Toy (Sandbox) 링크       │  ← Powder Toy 연결 섹션
└─────────────────────────────────────────────┘
```

### 3.3 게임 로직 (index.js) - 핵심 분석

#### 3.3.1 상태 변수

```javascript
let temp = 50;      // 온도 (0~100 범위)
let press = 50;     // 압력 (0~100 범위)
let score = 0;      // 생존 시간 (초)
let isGameOver = false;
let isPaused = false;
```

#### 3.3.2 게임 루프 (3개의 독립 타이머)

| 타이머 | 주기 | 역할 |
|--------|------|------|
| `gameInterval` | 100ms (10fps) | 물리 연산 (`updatePhysics`) |
| `scoreInterval` | 1000ms | 생존 시간 카운트 (`updateScore`) |
| `EventInterval` | 5000ms | 랜덤 교란 이벤트 (`triggerDisturbance`) |

#### 3.3.3 물리 엔진 (`updatePhysics`)

온도/압력 변화를 매 100ms마다 계산한다:

```
온도 변화량 = (스팀 × 0.05) - (리플럭스 × 0.04) - 자연감소(0.5) + 랜덤노이즈
압력 변화량 = (스팀 × 0.03) - (리플럭스 × 0.01) + 플러딩팩터 - 자연감소(2.0) + 랜덤노이즈
```

**플러딩 메커니즘**: 리플럭스 > 75%일 때 추가 압력 상승 발생
```javascript
if (reflux > 75) {
    floodingFactor = (reflux - 75) * 0.05;  // 최대 1.25 추가
}
```

**밸런스 분석**:
- 스팀 50%, 리플럭스 50% (기본값) 기준:
  - 온도: +2.5 - 2.0 - 0.5 = **0.0** (평형)
  - 압력: +1.5 - 0.5 - 2.0 = **-1.0** (자연 하락)
- **핵심**: 기본 상태에서 압력이 자연적으로 하락하므로, 플레이어는 지속적으로 스팀을 올리거나 리플럭스를 줄여야 함
- 리플럭스를 높이면 온도는 빨리 떨어지지만, 75% 초과 시 플러딩으로 압력이 급등

#### 3.3.4 교란 이벤트 시스템 (`triggerDisturbance`)

5초마다 20% 확률로 발생하는 랜덤 이벤트:

| 이벤트 | 온도 변화 | 압력 변화 | 위험도 |
|--------|----------|----------|--------|
| FEED FLOW SURGE | -5 | +8 | 중 (압력 급등) |
| COOLING WATER FAILURE | +10 | +10 | 고 (온도+압력 동시 급등) |
| STEAM PRESSURE DROP | -8 | -8 | 중 (온도+압력 동시 급락) |
| COMPRESSOR TRIP | 0 | +15 | 고 (압력 대폭 급등) |

이벤트 발생 후 3초간 경고 메시지가 표시된다.

#### 3.3.5 게임 오버 조건 & 사망 메시지

| 조건 | 사망 메시지 |
|------|------------|
| `temp >= 100` | "Column Runaway: Overheating!" |
| `temp <= 0` | "Reaction Quenched: Temp too low!" |
| `press >= 100` | "Catastrophic Overpressure: Explosion hazard!" |
| `press <= 0` | "Vacuum Implosion: Pressure loss!" |

#### 3.3.6 알람 시스템 (3단계)

| 범위 | LED 색상 | 상태 메시지 |
|------|---------|------------|
| 40~60 (안전) | 초록 | System Normal |
| 20~80 (경고) | 주황 | ANOMALY DETECTED |
| 0~10 또는 90~100 (위험) | 빨강 (깜빡) | CRITICAL WARNING |

### 3.4 CSS 디자인 (index.css)

- **디자인 테마**: 밝은 톤의 **글래스모피즘(Glassmorphism)** UI
- **폰트**: `Inter` (본문) + `Orbitron` (디지털 수치, 산업용 HMI 느낌)

  - `pulse`: 위험 LED 깜빡임
  - `flowBg`: 배관 내 유체 흐름 효과
- **게임 오버**: 글리치(glitch) 텍스트 효과, 블러 오버레이
- **증류탑 그래픽**: CSS로 구현된 탑체, 트레이(5단), 배관(피드/상단/하단)

## 4. ThePowderToyWeb - 상세 분석

### 4.1 개요

**The Powder Toy** 오픈소스 물리 샌드박스를 WebAssembly로 컴파일하고, **화학 공정용 커스텀 Lua 모드**를 자동 적용하여 브라우저에서 실행하는 서브 프로젝트이다.

### 4.2 WASM 빌드 (`powder_kokr.js` / `.wasm`)

- Emscripten으로 컴파일된 한국어 버전 The Powder Toy
- `create_powder_kokr()` 함수를 통해 WASM 인스턴스 생성
- canvas에 렌더링, WebGL 사용

### 4.3 Lua 자동 주입 메커니즘 (index.html)

```
1. luaCode 변수에 Lua 스크립트를 문자열로 정의
2. Module.preRun 단계에서 Emscripten 가상 파일시스템(FS)에 /autorun.lua 파일 생성
3. TPT 엔진이 시작 시 autorun.lua를 자동 감지/실행
4. 화학 공정 모드 활성화 완료
```

### 4.4 화학 공정 모드 (chemistry.lua) - 핵심 분석

#### 4.4.1 UI 커스터마이징 - 메뉴 정리

불필요한 카테고리를 숨겨 화학 공정에 집중:

| 숨기는 카테고리 | 카테고리 ID |
|----------------|------------|
| 생명체 (Life) | SC_LIFE (7) |
| 폭발물 (Explosive) | SC_EXPLOSIVE (2) |
| 방사능 (Radioactive) | SC_RADIOACTIVE (6) |
| 특수 (Special) | SC_SPECIAL (8) |
| 크래커 (Cracker) | SC_CRACKER (9) |

#### 4.4.2 커스텀 원소 등록

| 원소 | 코드 | 색상 | 타입 | 분자량 | 특성 |
|------|------|------|------|--------|------|
| **CH4** (메탄) | `CHEM_CH4` | 연두색 (0x90EE90) | 기체 | 15 | SMR 반응 주원료 |
| **CO** (일산화탄소) | `CHEM_CO` | 파란색 (0x6464FF) | 기체 | 28 | SMR 반응 생성물 |
| **H2** (수소) | `CHEM_H2` | 분홍색 (0xFFC8FF) | 기체 | 2 | SMR 반응 생성물, 가연성(100) |
| **NI** (니켈 촉매) | `CHEM_NI` | 은색 (0xC0C0C0) | 고체 | - | 촉매 역할, 높은 열전도율(251) |

#### 4.4.3 화학 반응 로직 - Steam Methane Reforming (SMR)

실제 산업 공정인 **수증기 메탄 개질(SMR)** 반응을 시뮬레이션:

```
CH₄ + H₂O → CO + 3H₂  (T > 700°C, Ni 촉매)
```

**구현 로직**:

```
매 프레임마다 CH4 입자에 대해:
  1. 온도 > 973K (700°C) 확인
  2. 인접 8칸에 수증기(WTRV) 존재 확인
  3-a. 반경 2칸 내에 NI(니켈 촉매) 존재 → 반응 진행
  3-b. NI 없어도 2% 확률로 자연 반응 (비촉매 반응)
  4. 반응 시: CH4 → CO, H2O → H2, 온도 100K 감소 (흡열 반응)
```

**화학적 정확성**:
- 실제 SMR은 흡열 반응 → 온도 감소 반영됨
- 촉매(Ni) 존재 시 반응 확률 대폭 증가 → 현실적
- 973K 반응 온도 → 실제 산업 조건(700~1000°C)과 부합

### 4.5 로컬 서버 구성

#### `run_server.py` (간단 서버)
- Python HTTP 서버 (포트 8000)
- **COOP/COEP 헤더** 추가 (SharedArrayBuffer 활성화 → WASM 멀티스레딩에 필수)

#### `serve-wasm.py` (SSL 서버, 원본 TPT 도구)
- 자체 서명 SSL 인증서 자동 생성 (OpenSSL)
- HTTPS로 서빙 (보안 컨텍스트 필요)
- 원본 TPT의 meson 빌드 시스템과 연동

#### `vercel.json` (배포 설정)
- Vercel 배포 시 COOP/COEP 헤더 자동 적용

## 6. 기술적 특징 분석

### 6.1 장점

| 항목 | 설명 |
|------|------|
| **산업 현실성** | 실제 HMI 화면과 유사한 UI, 실제 화학 공정 용어(TR-101, PI-102, TV-101A) 사용 |
| **물리 모델** | 스팀/리플럭스 상호작용, 플러딩 현상, 자연 감쇠 등 현실적 모델링 |
| **교란 시스템** | 실제 플랜트에서 발생하는 사고 유형 반영 (피드 서지, 냉각수 고장 등) |
| **화학 정확성** | SMR 반응의 온도 조건, 촉매 효과, 흡열 반응 등 정확히 구현 |
| **디자인 품질** | 글래스모피즘, 산업용 폰트, LED 알람 등 높은 완성도의 UI |
| **기술 통합** | Vanilla JS 게임 + WASM 샌드박스 + Lua 스크립팅의 3중 레이어 |

### 6.2 게임 밸런스 분석

**난이도 곡선**:
- 초기: 양쪽 50%에서 온도 평형, 압력만 자연 하락 → 쉬움
- 중기: 교란 이벤트가 축적되면서 밸런스 깨짐 → 중간
- 후기: COMPRESSOR TRIP(+15 압력) 연속 발생 시 즉사 가능 → 어려움

**밸런스 이슈**:
- 기본 상태에서 압력이 매 틱 -1.0씩 떨어져서, 스팀을 꾸준히 올려야 함
- 리플럭스 75% 초과 시 플러딩으로 급격한 압력 상승 → 위험 트랩
- COOLING WATER FAILURE(+10/+10)이 두 번 연속 오면 거의 게임 오버

## 7. 두 게임 간 관계

```
┌─────────────────────────────────────┐
│     증류탑 서바이벌 (index.html)       │
│     "운전(Operation)" 시뮬레이션      │
│                                      │
│   [🚀 LAUNCH POWDER TOY] 버튼        │
│            │                         │
└────────────┼─────────────────────────┘
             ▼
┌─────────────────────────────────────┐
│   ThePowderToyWeb (새 탭 열림)       │
│   "현상(Phenomena)" 시뮬레이션        │
│   - SMR 반응을 입자 단위로 관찰       │
│   - 열, 압력, 화학 반응의 미시적 동작  │
└─────────────────────────────────────┘
```

- **증류탑 서바이벌**: 거시적(Macro) 관점 - 공정 운전자의 시점에서 밸브 조작
- **Powder Toy**: 미시적(Micro) 관점 - 분자 수준에서 화학 반응 관찰
- 두 게임은 **보완적 관계**로, 같은 화학 공정을 다른 스케일에서 체험

## 8. 코드 품질 평가

| 항목 | 평가 | 비고 |
|------|------|------|
| 코드 구조 | B+ | 단일 파일이지만 함수 분리가 잘 되어 있음 |
| 가독성 | A | 변수/함수 이름이 직관적, 주석 적절 |
| UI/UX | A | 산업용 HMI 느낌 잘 살림, 시각 피드백 우수 |
| 게임 밸런스 | B | 압력 자연 하락이 강해서 스팀 위주 플레이 유도됨 |
| 확장성 | B- | 단일 JS 파일, 모듈화 시 확장 가능 |
| 보안 | A | 순수 프론트엔드, 외부 통신 없음 |
| 배포 준비도 | A- | Vercel 설정 포함, COOP/COEP 헤더 완비 |

## 10. 요약

이 프로젝트는 **화학공학 교육/체험 목적**의 게임으로, 증류탑 운전 시뮬레이션(서바이벌 게임)과 분자 수준 화학 반응 시뮬레이션(Powder Toy 모드)을 결합했다. 실제 산업 현장의 HMI 화면을 모방한 UI, 현실적인 물리 모델, 그리고 정확한 SMR 반응 구현이 특징이다. The Powder Toy의 한국어 WASM 빌드에 커스텀 화학 원소(CH4, CO, H2, Ni)를 Lua 자동 주입 방식으로 추가한 점이 기술적으로 독창적이다.
